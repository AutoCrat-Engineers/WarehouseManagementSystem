# WMS UI Standardization & Bug Fixes Changelog

## 1. Eliminated Full-Page Re-Rendering Flickering
- **Issue**: Navigating pages or typing in a search bar would cause the entire module to unmount, replaced by a full-screen white `ModuleLoader`, ruining the UX and making the app feel slow.
- **Fix**: Implemented a robust `initialLoad` (boolean) guard across all dashboard modules. The full-page skeleton loader now explicitly runs ONLY once when the module mounts. 
- **Modules Fixed**:
  - `ItemMasterSupabase.tsx`
  - `InventoryGrid.tsx`
  - `DispatchSelection.tsx`
  - `PalletDashboard.tsx`
  - `PackingDetails.tsx`

## 2. Standardized Item Master Pagination & Load Flow
- **Issue**: `ItemMaster` was downloading the entire database into memory and filtering using internal JS `useMemo()` hooks instead of the database.
- **Fix**: Rewrote the entire data flow to mirror modern WMS standards:
  - Switched from local `.slice()` to server-side Postgres `.range(limit, offset)`.
  - Stats for the 3 summary cards now use lightweight `head:true` counting queries.
  - Implemented the spinning `RefreshButton` component inside the Action Bar to provide precise user feedback on network refresh.
  - CRUD operations (Edit/Create/Delete) now safely call `fetchCounts()` and `fetchData()` in the background to keep pagination accurate.

## 3. Fixed Sticker Generation Search Engine (PackingModule.tsx)
- **Issue**: Typing into the Sticker Generation search bar did nothing. 
- **Fix**: Hooked the `searchTerm` up to the API hook dependency array. 
- **Issue 2**: Users wanted to search for Movement Number, Item Name, Part Number, and Master Serial Number. But the data lives dynamically split across two database tables.
- **Fix 2**: Wrote a dynamic cross-table query pipeline that:
  - First queries the `items` table for matching names/parts.
  - Extracts the UUIDs.
  - Safely injects them into the primary `packing_requests` API call alongside the `movement_number` filter so users can find items by scanning ANY related barcode or string. 
- **Robustness**: Replaced all string templates inside PostgREST filter queries with explicit regex `.replace(/"/g, '\"')` to ensure special characters (like commas or quotes inside W/O numbers) do not crash the database parsing engine.
